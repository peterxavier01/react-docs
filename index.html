Introduction

React is one of the most popular JavaScript libraries used for front-end development. It was developed by Facebook.


Unlike other libraries which manipulate the DOM, React possesses what is called a Virtual DOM which allows it update only the parts of the website that have changed. This drastically increases the speed of updates


React is open source and it also has a very large community. It is maintained by the community and Facebook.


To get started with React, ensure that you have Nodejs installed on your local computer. To setup a new React Project, Facebook has provided a simple tool called Create React App


To create a new React app with Create-React-App, run the following command in the Terminal to create a React app called "first-app"

    
        npx create-react-app first-app
        cd first-app
        npm start

This will install the required dependencies, then configure and start the app on localhost:3000

Components

Components in React allow a webpage to be split into separate and reuseable parts. For example, a navbar can be taken as a component to be reused on the various pages of a particular website. A button or a search bar could also be taken as components as the case may be.


The idea of components is derived from a programming concept called Separation of concerns which states that each concern should be separated into individual pieces.


In React, there are two types of components:


  • Functional components
  • Class components

    • A functional component is simply a JavaScript function; although its name must begin with capital letter, else the browser will treat it like an HTML element instead of a component.


    An example of a functional component is shown below:

        
        function Greeting() {
            return <h1>Hello world.</h1>;
        }

    The code above defines a functional component called Greeting, which returns a simple React element.


    On the other hand, class components extend the React.Component class and are used for more advanced user interactions like animations, forms, e.t.c.


    Rewriting the Greeting functional component as class component:

        
        class Greeting extends React.Component {
            render() {
                return <h1>Hello world.</h1>;
            }  
        }

    Class components require a render method because it determines what is shown on a webpage.

    In rendering components, we need to create a corresponding JSX element. For example:

        
        const element = <Greeting />;

    Now, we can render our component on the page using our user-defined element: 

        
        function Greeting() {
                return <h1>Hello world.</h1>;
        }
        const element = <Greeting />;
        ReactDOM.render(
            element,
            document.getElementById('root')
        );
    JSX

    JSX is a syntax extension to JavaScript that enables us to build UI elements right in the JavaScript code.


    It should be noted that React does not require JSX. It is just a common practice in the community to use JSX as it eases user interface development. It also allows React to show useful warnings and error messages.


    The code below shows how a JSX syntax looks like:

        
        ReactDOM.render(
            <h1>Hello world.</h1>;
            document.getElementById('root')
        );

    <h1>Hello world.</h1> is the JSX element.


    We can write any JavaScript expression inside JSX using curly braces.


        
        const car = "Mercedes";
        const element = <p>I have the latest {car}.</p>;

        ReactDOM.render(
            element,
            document.getElementById('root')
        );
    Props

    Similar to JavaScript functions, functional components can accept arguments called props, which represent an object.


    We can use props in our Greeting component as shown below:

        
        function Greeting(props) {
            return <p>Hello, {props.name}.</p>;
        }

    We can now add a name attribute to our element:

        
        const element = <Greeting name="Mike" />;

    Note the following:


  • An element can possess multiple custom attributes which will be passed to the component using the props object.
  • Also, you can assign custom names to your attributes
  • Props are read-only i.e. components cannot modify their props.

    • Class components also use props but in a slightly different way:

        
        class Greeting extends React.Component {
            render() {
                return <p>Hello {this.props.name}.</p>;
            }  
        }

    Class components access props by using this.props.

    State

    Alot of web apps require their components to change their data, for example, typing in a input field, clicking a button,etc. This is where State comes into the picture. State allows components to manage and change their data. It is an object added as a property in class components.


        
        class Greeting extends React.Component {
            state = {
                name: "Dave"
            }

            render() {
                return <h2>Hello {this.state.name}.</h2>;
            }  
        }

    From the code above:


  • It can be seen that state is just a simple object containing key:value pairs.
  • Also, similar to props, the values can be accessed using this.state.
  • When the component renders, the state will be initialized with the given value and a heading that reads "Hello Dave".
  • Also, note that state object can contain multiple key:value pairs, separated by commas as in a regular JavaScript object.

    • State should not be modified directly. React provides a setState() method for modifying state.

        
        this.setState({
            name: "Mike",
            age: 30
        });

    You need to pass an object with the new key:value pairs to the setState method.


    When setState is called, React automatically re-renders the affected component with the new state.


    The code below shows the setState method in action:


        
        class Counter extends React.Component {
            state = {
                counter: 0;
            }
            increment = () => {
                this.setState({
                    counter: this.state.counter + 1
                });
            }
            render() {
                return <div>
                <p>{this.state.counter}</p>
                <button onClick = {this.increment}>Increment</button>
                </div>;
            }
        }

    Components that use state are known a stateful, while components that do not use state are known as stateless.

    Hooks

    Earlier versions of React only allowed the use of state with class components. Recent versions introduced a new feature called hooks which allows the use of state with functional components.


    The hook that allows the use of state in functional components is called the useState hook. To use it, we first need to import it:

        
        import React, { useState } from "react";

    useState returns the current state value and a function, that lets you change the state. Also, useState takes one argument, which is the initial value of the state.


    See the useState hook in action:

        
        function Greeting() {
            const [name, setName] = useState("Mike");

            return <h2>Hello {name}.</h2>;
        }

    From the example above:


  • We create a name state variable and a setState function.
  • The square brackets syntax is known as array destructuring. This assigns the name variable to the current state value and setName to the function that allows the change of state.
  • The variables can be given any names of your choice.
  • We then pass "Mike" as the initial value for our name variable to useState().
  • Lastly, you can create multiple state variables with their corresponding set methods.

    • We can rewrite the counter app example given in the state section using functional components and hooks, instead of class components:

        
        function Counter() {
            const [counter, setCounter] = useState(0);

            function increment() {
                    setCounter(counter + 1);
            }
            
                return <div>
                <p>{counter}</p>
                <button onClick = {increment}>Increment</button>
                </div>;
            }
        }

    It can be observed that, compared to class components, the code is more concise and easier to read and understand.


    Finally, keep in mind that hooks can only be used inside functional components.

    Event Handling

    Event handling in React is almost the same as handling events in the DOM. The only difference is that the event handler needs to be passed in curly braces and the event names are in camelCase.

        
        <button onClick = {handleClick}>Click Me</button>

    Click the button will call the handleClick function.


    Users interact with web pages in various ways and one of them is by inputting in text fields. User input in React is handled using the onChange event of the text field.


    When there is a change in the value of the text field, the event handler is called, thereby updating the value of the text field in the state. This allows one to always have the actual value of the text field in the state.


    Let's see we handle user input in React:

        
        function Converter() {
            const [mm, setMm] = useState(0);

            function handleChange(e) {
                setMm(e.target.value);
            }

            function convert(mm) {
                return (mm/1000).toFixed(3);
            }

            return <div>
                    <input type="text" 
                            value={mm}
                            onChange={handleChange}
                    />
                    <p> {mm} mm is {convert(mm)} metres </p>
                    </div>
        }

    The Converter component above:


  • Includes a text field that calls the handleChange function when its value changes.
  • The handleChange function updates the state with the current value of the text field, which results in the re-rendering of the component and displays the corresponding metre value.
  • The value of the text field is accessed through the e object, which is the React event. It is passed to the event handler function as an argument and used to access the event object.

    • When a form is submitted, the default behaviour is to reload the page. To prevent this behaviour, we can use the following lines of code in our handleSubmit function:

        
        function handleSubmit(e) {
            //Some lines of code
            e.preventDefault();
        }

    The e.preventDefault() statement is what prevents this default behaviour.

    Lifecycle Methods

    Some unique methods are provided by React for class components which are called when components are mounted, unmounted or updated. These methods are called Lifecycle Methods


    Mounting is the process whereby a component is rendered on the page. Whereas, Unmounting is the process whereby a component is removed from the page.


    Some of these lifecycle methods are:


  • componentDidMount()
  • componentWillUnmount()
  • componentDidUpdate()

    • The componentDidMount method is called when a component is rendered on the page. For example, we can use it in our Counter app to set the initial value of the counter:

        
        componentDidMount() {
            this.setState({counter: 20});
        }

    This will set the initial value of the counter when the component is rendered.


    The componentWillUnmount() lifecycle method is called right before the component is removed from the DOM. It can be used to free up resources taken by the component.


    The componentDidUpdate() lifecycle method is called when a component is updated in the DOM. For example, we can alert the current value of the counter app after each increment:

        
        componentDidUpdate() {
            alert("Counts: " + this.state.counter);
        }

    componentDidUpdate() is only called when a component is updated.


    The lifecycle methods just covered are only available for class components. However, React provides a hook called useEffect, which makes lifecycle methods available to functional components. It is a combination of the componentDidMount, componentWillUnmount and componentDidUpdate methods.


    Using the useEffect hook in place of the lifecycle methods in our counter app:

        
        useEffect(() => {
            alert("Counts: " + counter);
        });

    You will notice that the alert box pops during the initial rendering of the page. This is because useEffect runs both during the first render and after every update.


    To call the method only when there are changes, we will introduce a second argument known as a dependencies array:

        
        useEffect(() => {
            alert("Counts: " + counter);
        }, [count]);

    Now, the useEffect() method will only run when the count changes.


    Note that the useEffect hook has to be imported just like the useState hook:

        
        import React, { useState, useEffect } from "react";
    Rendering Lists

    Repeating elements such as lists or sections are usually seen in web apps, whereby the same DOM element is repeated with a different data set.


    Let look at an array of names:

        
        const names = ["Mike", "Dave", "Jude"];

    We need to render a list <li> element for each item in the array.


    We can define a NamesList component and pass it the array as a prop using a custom data attribute:

        
        <NamesList data={names} />

    The array is now accessible via props and hence, we can write the component logic:

        
        function NamesList(props) {
            const names = props.data;
            const listItems = names.map((value) => 
            <li>{value}</li>
            );
            return <ul>{listItems}</ul>
        }

    From the above lines of code:


  • We take the names array from the props, loop through the array using the JavaScript map method and return a <li> element for each item.
  • The resulted array is stored in the listItems variable.
  • Then, the component returns the listItems array inside a <ul> tag.

    • This code will result in a warning saying that each element needs a unique key.


    Each element in a list requires a key attribute. This is because keys act as unique identifiers, helping React to understand which items have changed, added or removed.


    Keys could be IDs from your data or auto-generated indexes:

        
            const listItems = names.map((value, index) => 
            <li key={index}>{value}</li>
            );
    React Router

    To get started with the React Router, we first need to install it by running:

        
        npm install react-router-dom@6

    This installs version 6 of the React Router.


    If you are using yarn, then run:

        
        yarn add react-router-dom@6

    After installation, the first thing to do is to make React Router available anywhere in your app. To do this, import BrowserRouter from react-router-dom and then wrap the root component in it like so:

        
        import React from 'react';
        import ReactDOM from 'react-dom';
        import App from './App';
        import { BrowserRouter } from 'react-router-dom';

        ReactDOM.render(
            <BrowserRouter>
                <App />
            </BrowserRouter>
            document.getElementById('root');
        );

    To route to other components, you need to have multiple components like Home, About and Contact


    Since the App component acts as the root component, we will be creating all our routes in it.

        
        import { Routes, Route } from 'react-router-dom';
        import Home from "./Home";
        import About from "./About";
        import Contact from "./Contact";

        function App() {
            return (
                <div className="App">
                    <Routes>
                    <Route path="/" element={ <Home/> } />
                    <Route path="about" element={ <About/> } />
                    <Route path="contact" element={ <Contact/> } />
                    </Routes>
                </div>
            );
        }

        export default App;

    From the above lines of code:


  • Routes acts as a container for all the individual routes that will be created in the app.
  • Route is used to create a single route. It takes in two attributes: path and element.
  • path specifies the URL path of the desired component. You can call this pathname anything you want.
  • The path with a backslash / means that the component will be rendered first when the app loads for the first time.
  • element specifies the component the route should render.

    • To navigate to other routes and pathnames, we will use a different React feature called Linkin desired components.

        
        import { Link } from 'react-router-dom';
        
        function Home() {
            return (
                <div>
                    <h1>This is the home page</h1>
                    <Link to="about">Click to view the about page</Link>
                    <Link to="contact">Click to view the contact page</Link>
                </div>
            );
        }

        export default Home;

    From the above lines of code:


  • The Link component is similar to the anchor element <a> in HTML.
  • Its to attribute specifies the path which the link takes you to.
